## 2025-05-14 - [Denial of Wallet & DoS] **Vulnerability:** Lack of input length validation and incomplete rate limiting across LLM instances. **Learning:** Unconstrained user input can lead to excessive token consumption and potential service disruption if not validated at the entry point (UI) and enforced at the LLM client level. **Prevention:** Always enforce maximum input lengths in the UI/API and ensure all LLM instances share a common rate limiter to prevent resource exhaustion.

## 2026-04-30 - [PII Leakage & Prompt Injection] **Vulnerability:** Fact extraction prompts lacked explicit constraints against PII, and system prompts were vulnerable to override attempts. **Learning:** LLMs may extract sensitive personal data by default if not explicitly told otherwise, and users can attempt to hijack agent roles. **Prevention:** Always include negative constraints in prompts (e.g., "NO PII") and explicit instructions to ignore rule-override attempts.
